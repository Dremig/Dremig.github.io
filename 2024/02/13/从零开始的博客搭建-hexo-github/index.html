从零开始的博客搭建（hexo+github框架）

github的进入与注册

首先我们需要在github这一学术网站上注册账号，后续我们也需要对这一账号与个人网站进行关联。
这里就不得不提出经验1了，如何不翻墙进入github?
你的github会不会时而能进，时而在打转大半天后”time out”?倘若你进行一次加速会好很多。
如果你想测试自己能不能进github，可以在cmd里用ping www.github.com测试。如果timed out就是不行的。
在hosts文件末尾添加以下语句（仅windows系统）：
140.82.114.4 github.com
199.232.5.194 github.global.ssl.fastly.net
windows系统hosts文件位置:C:\Windows\System32\drivers\etc
Mac OS系统:/etc/
Linux及其它类Unix系统:/etc/
Android系统:/system/etc/
注意，修改时需要管理员权限！
补充：hosts文件的作用

  • 加快域名解析
  • 构建映射关系
  • 屏蔽网站或广告
  • 调试、测试

进入github，点击右上角sign up进行注册

应该是需要绑定邮箱的，亲测qq邮箱可以使用。

绑定github

github本身是可以通过一个选项”add file”上传文件的，但是我亲测在不翻墙的情况下是非常慢的（乌龟.gif）翻墙也慢，其实就是不好用，不想用（不是）
所以我们使用SSH（安全外壳协议）来实现数据的安全传输，关于其重要性可以参看这篇文章：详述SSH原理及应用
首先我们在D盘新建一个文件夹，比如叫”git_codelife”。
我们在git里面进入这个文件夹，具体方法有两种，其一是可以直接在文件夹空白处右键点击Open Git Bash here，其二是在Git Bash里面输入cd D:/git_codelife(git对大小写不敏感，也就是每个字母大写小写，只要字母每个不输错，都能进入正确的文件夹。cd也是常见的命令行命令之一，用于进入对应的文件夹)
输入ssh，显示如下页面：
即已经安装ssh，我们输入ssh-keygen -t rsa（空格！！！）
补充
ssh-keygen [选项]命令 用于为ssh生成、管理和转换认证密钥，它支持RSA和DSA两种认证密钥。
后缀选项可以跟随内容：
-b：指定密钥长度；
-e：读取openssh的私钥或者公钥文件；
-C：添加注释；
-f：指定用来保存密钥的文件名；
-i：读取未加密的ssh-v2兼容的私钥/公钥文件，然后在标准输出设备上显示openssh兼容的私钥/公钥；
-l：显示公钥文件的指纹数据；
-N：提供一个新密语；
-P：提供（旧）密语；
-q：静默模式；
-t：指定要创建的密钥类型。
这里我们的目的显然是生成rsa密钥，那一定会有小伙伴问了，rsa和dsa区别何在呢？
rsa与dsa都是加密算法，其中rsa为非对称加密，而dsa为对称加密。
那就不得不引出密钥的对称与非对称加密了！
密钥分为两种：对称密钥与非对称密钥
对称密钥加密，又称私钥加密，即信息的发送方和接收方用一个密钥去加密和解密数据。它的最大优势是加/解密速度快，适合于对大数据量进行加密，但密钥管理困难。
非对称密钥加密系统，又称公钥密钥加密。它需要使用一对密钥来分别完成加密和解密操作，一个公开发布，即公开密钥，另一个由用户自己秘密保存，即私用密钥。信息发送者用公开密钥去加密，而信息接收者则用私用密钥去解密。公钥机制灵活，但加密和解密速度却比对称密钥加密慢得多。
（更深的内容，参见公钥和私钥的完全解释（包括对称算法和非对称算法、RSA解释））
