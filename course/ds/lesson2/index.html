    Chapter2 : The Relational Model

    • relationship: 实体与实体之间存在的联系

    • relation: 数学概念，体现了这个表的表结构，也可以直接说就是表。

    Structure of Relational Databases

    Basic Structure

    使用笛卡尔积的概念：given sets \(D_1\ D_2\ ...\ D_n\)，那么relation是\(r = D_1\ ×D_2\ ×...\ D_n\)。

    我们看到笛卡尔积实际是涵盖了表的内容的，所以我们说关系是反映了特定意义的子集。

    Attribute Types

    • 每个属性都有其名称，每个属性允许的范围叫做其域(domain)。
    • 属性的值被要求是原子性的，即不可分割。因此多值属性值、复合属性值不是原子值。
    • null是每个域的成员，其复杂性我们后续讨论。

    Concepts About Relations

    一个关系涉及两个概念：关系模式与关系实例

    • 关系模式(relation schema)描述了关系的结构。
    • 关系实例(relation instance)对应某一时刻关系的具体内容(关系中数据的快照)。

    e.g. 变量：关系；变量类型：关系模式；变量值：关系实例。

    Relation Schema

    假设\(A_1\ A_2 ...\ A_n\)是一系列属性，那么关系模式就是\(R = (A_1\ A_2 ...\ A_n)\)，\(r(R)\)代表基于该关系模式的关系。

    Relation Instance

    关系当前的值(即关系实例)由表实现，关系r的一个元素t是一个元组，用表中一行表示，表示其属性name，使用t[name]即可。

    The properties of Relations

    • 元组的顺序是无关紧要的（即，元组可以以任意顺序存储）
    • 关系中没有重复的元组。
    • 属性值是原子性的。

    Key

    设 K ⊆ R:

    • K 是 R 的超码（superkey），如果 K 的值足以唯一标识每个可能关系 r(R) 中的一个元组。例如，{ID} 和 {ID, name} 都是关系 instructor 的超码。
    • K 是候选码（candidate key），如果 K 是最小的超码。例如，{ID} 是关系 instructor 的候选码，因为它是超码且没有任何子集可以代替它。
    • K 是主码（primary key），如果 K 是候选码且由用户明确指定。主码通常通过下划线标记。
    • r(A, B, C)，s(B, D)，我们可以说关系 r 中的属性 B 是引用了s的外键，而 r 是参照关系，s 是被参照关系。
      • 注意被引用的属性一定是另一个表的主键。

    就外键我们可以画出schema diagram.

    Query Language

    • 关系代数（Relational Algebra） —— SQL的基础
    • 元组关系演算（Tuple Relational Calculus）
    • 域关系演算（Domain Relational Calculus） —— QBE（查询-by-example）

    Fundamental Relational-Algebra Operations

    六个基础操作符：

    • select 选择
    • project 投影
    • union 并
    • set difference 差
    • Cartesian product 笛卡儿积
    • Rename 改名（重命名）

    "The operators take one or two relations as inputs, and return a new relation as a result."

    select

    输入选择的条件，输出元组。

    符号：\(σ_p(r)\)， p称为选择谓词(selection predicate)。

    很简单啦，就是p是与或非组成的，取能使得其为真的元组。

    project

    也是比较基础的操作，取属性子集，然后去掉重复的元组。

    符号：\(\Pi_{A_1\ A_2 ...\ A_n}(r)\)，\(A_1, A_2, ..., A_n\)是属性子集。

    union

    符号：\(r \cup s\)，r和s的属性集合必须相同。

    set difference

    符号：\(r - s\)，r和s的属性集合必须相同。

    Cartesian product

    穷举所有元组的组合结果。

    符号：$r × s\ \ r × s = {{t\ q} | t ∈ r \ and\ q ∈ s} $

    • Set intersection 交
    • Natural join 自然连接
    • Division 除
    • Assignment 赋值

    Set intersection

    Natural join

    符号：\(r ⨝ s\)。

    怎么忽然一大串呼我脸上，吓死我了。

    \(r ⨝ s = \Pi_{r.A\ r.B\ r.C\ r.D\ s.E}\ σ_{r.B = s.B ∩ r.D = s.D}(r × s)\)

    考虑关系 \( r \) 中的每一对元组 \( tr \) 和关系 \( s \) 中的每一对元组 \( ts \)。如果 \( tr \) 和 \( ts \) 在 \( R \cap S \) 中的每一个属性上具有相同的值，则将一个元组 \( t \) 添加到结果中，其中：

    • \( t \) 在关系 \( r \) 中与 \( tr \) 具有相同的值，
    • \( t \) 在关系 \( s \) 中与 \( ts \) 具有相同的值。

    其实就是找相同元素值还相同的元组，然后排列组合合并。

    Division

    看起来也是很复杂，其实就是所有s中的元组都能在r中找到配对，则取a。

    举个例子。

    公式：\((r \div s = {t | t ∈ \Pi_{R-S}(r) ∩ [∀ u ∈ s (tu ∈ r)]})\)

    再来个例子。

    性质/特点

    如果\(r \div s = q\)，那么q是最大的关系使\(q × s = r\)。

    Assignment Operation

    \(temp1 ← π_{R-S}(r)\\ temp2 ← π_{R-S}((temp1 × s) – π_{R-S,S}(r))\\ result = temp1 – temp2\)

    \(←\)将右赋值到左。

    Modification of the Database

    可以使用以下操作修改数据库的内容：

    • 删除(Deleting)
    • 插入(Inserting)
    • 更新(Updating)

    所有这些操作都通过赋值操作符来表达。

    Delete

    删除请求的表达方式类似于查询，不同之处在于，删除操作不会将元组显示给用户，而是将选中的元组从数据库中删除。

    删除操作只能删除整个元组，不能只删除某些特定属性的值。

    在关系代数中，删除操作表示为：\(r←r−E\)。其中，r 是一个关系，E 是一个关系代数查询。
